嵌入式内容相关组件

<script>

<script> 标签用于嵌入客户端脚本（JavaScript）。

  • async: 指定脚本在下载时与页面解析并行进行，脚本下载完毕后立即执行。
  • crossorigin: 设置请求的 CORS 模式。
  • defer: 指定脚本在下载时与页面解析并行进行，但延迟执行直到页面解析完毕。
  • integrity: 允许浏览器检查下载的脚本文件，以确保文件未被篡改。
  • nomodule: 指定脚本不应在支持 ES2015 模块的浏览器中执行。
  • referrerpolicy: 指定获取脚本时发送的引用信息。
  • src: 指定外部脚本文件的 URL。
  • type: 指定脚本文件的 MIME 类型。

可以通过 <script> 标签嵌入脚本或引用外部脚本文件，用于网页交互和功能扩展。

<noscript>

<noscript> 标签用于处理禁用了浏览器脚本或不支持客户端脚本的用户情况。

通常在需要向用户显示替代内容或信息时使用。

<embed>

<embed> 标签定义了一个容器，用于嵌入外部资源，如网页、图片、媒体播放器或插件应用程序。

  • height: 指定嵌入内容的高度。
  • width: 指定嵌入内容的宽度。
  • src: 指定嵌入内容的 URL。
  • type: 指定嵌入内容的 MIME 类型。

对于图片、音视频等内容的嵌入，推荐使用更适合的标签如 <img>、<video> 或 <audio>。

<object>

<object> 标签定义了一个容器，用于包含外部资源，如网页、图片、媒体播放器或插件应用程序。

  • data: 指定 <object> 元素所引用资源的 URL。
  • form: 指定 <object> 元素所属的表单。
  • height: 指定嵌入内容的高度。
  • width: 指定嵌入内容的宽度。
  • name: 指定 <object> 元素的名称。
  • type: 指定嵌入内容的 MIME 类型。
  • typemustmatch: 指定是否要求 <object> 元素的 type 属性与实际资源的 MIME 类型匹配。
  • usemap: 指定 <object> 元素要使用的客户端图像映射的名称。

用于嵌入各种外部资源，但大多数浏览器已不再支持插件类型的内容，建议使用更现代化的替代方案。

<param>

<param> 标签用于为 <object> 元素提供参数，通常用于插件或嵌入式应用程序的配置。

  • name: 参数名称。
  • value: 参数值。

与 <object>元素配合使用，提供初始化参数或配置信息。
